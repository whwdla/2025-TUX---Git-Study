#### 1. 브랜치란
**버전 관리의 시작** | 깃으로 버전 관리를 시작하면 기본적으로 main 브랜치가 만들어진다. 사용자가 커밋할 때마다 main 브랜치는 어떤 게 최신 커밋인지 정보를 가진다.

**새 브랜치를 만든다면?** | 기존 파일은 main 브랜치에 유지하며 새 브랜치에서 기존 파일 내용을 수정하거나 새로운 기능을 추가한다. 이를 '분기(branch)한다'라고 한다.

**원하는 작업을 마쳤을 때** | 분기했던 브랜치를 main 브랜치에 합치는 것을 '병합(merge)한다'고 한다.

### 2. 브랜치 만들기 및 이동하기
**새 브랜치 만들기** | 아래의 명령을 사용하여 브랜치를 만들거나 확인 할 수 있다.
    
    $ git branch //브랜치 목록 확인
    $ git branch [이름] // 새 브랜치 생성

**현재 작업하고 있는 브랜치 확인하기** | ***$ git log*** 명령을 사용하였을 때 (HEAD ->)가 가르키고 있는 브랜치가 현재 작업하고 있는 브랜치이다. 혹은, 파일 경로 끝에 (main)과 같이 현재 작업 중인 브랜치가 표시된다.

    $ git log --oneline //해당 명령을 통해 log를 한 줄에 하나씩 볼 수 있다.

**브랜치 전환하기 - git switch** | 브랜치가 여러 개라면 서로 다른 커밋을 만들 수 있다. 이를 위해선 브랜치를 전환하는 과정이 필요하다.

    $ git switch [이름] //[이름]에 해당하는 브랜치로 전환한다.

### 3. 브랜치 정보 확인하기
**브랜치와 커밋 관계 알아보기** |

    $git log --branches //해당 옵션을 통해 브랜치마다 최신 커밋을 한 눈에 볼 수 있다.
    $git log --graph //해당 옵션을 통해 브랜치와 커밋 관계를 더 보기 쉽게 그래프의 형태로 볼 수 있다.

**브랜치 사이의 차이점 살펴보기** | 아래의 명령을 사용하여 마침표 왼쪽에 있는 브랜치를 기준으로 오른쪽 브랜치와 비교한다. 왼쪽 브랜치에는 없고 오른쪽 브랜치에 있는 커밋만을 보여준다.

    $ git log [이름1]..[이름2]


### 4. 브랜치 병합하기
**서로 다른 파일 병합하기** | 

    1. main 브랜치로 전환한다.
    2. git merge 명령 뒤에 가져올 브랜치 이름을 적는다.
    3. ls -al 명령을 통해 새 브랜치에 있던 파일이 main 브랜치에 합쳐져 있음을 확인한다.

**서로 다른 브랜치에서 한 문서의 다른 부분을 수정했을 때 병합하기** |

    1. main 브랜치로 전환한다.
    2. git merge 명령을 통해 새 브랜치를 main 브랜치로 끌어온다.
    3. 새 브랜치에서의 수정 내용이 자연스럽게 합쳐진다.

**서로 다른 브랜치에서 한 문서의 같은 부분을 수정했을 때 병합하기** |

    1. main 브랜치로 전환한다.
    2. git merge 명령을 통해 새 브랜치를 main 브랜치로 끌어온다.
    3. 다른 경우처럼 빔이 열리지 않고 경고 메세지가 나타난다. txt 파일의 경우 각 브랜치의 내용이 가운데 가로줄 사이로 둘 다 나타난다.

**병합이 끝난 브랜치 삭제하기** |

        $ git branch -d [이름] //해당 명령을 main 브랜치에서 사용하여 [이름]에 해당하는 브랜치를 삭제한다.

**cherry-pick으로 병합하기** | 만약, 진행 상황이 같은 두 브랜치가 있을 때, 두 브랜치를 병합하면 새로운 버전이 생겨날 것이다. 이에 반해 cherry-pick은 브랜치 전체를 합치는 것이 아니라, 브랜치 중 특정 버전의 변경 내용만 합치려고 할 때 사용하는 기능이다.

    1. $ git switch [이름] //병합될 브랜치로 이동한다.
    2. $ git cherry-pick [체리픽할 해시] //체리픽할 커밋 해시에 해당하는 버전을 가져와서 병합한다.

### 1. 원격 저장소와 깃허브
**원격 저장소란?** | 자신의 컴퓨터에서 작업한 뒤 컴퓨터 안에 저장할 때의 저장 장소를 지역 저장소(local repository)라고 한다. 작업한 내용을 지역 저장소에만 저장하는 것은 안전하지 않음으로 지역 저장소와 원격 저장소(remote repository)를 연결해서 쉽게 백업할 수 있다.

**깃허브로 할 수 있는 일들**

1. **원격 저장소에서 깃을 사용할 수 있다.**

    깃허브는 깃 사용을 위한 원격 저장소를 제공하는 서비스이므로 따로 깃을 설치하지 않고도 온라인에서 깃의 버전 관리 기능을 사용할 수 있다.

2. **지역 저장소를 백업할 수 있다.**

    깃허브에 원격 저장소를 만들고 사용자 컴퓨터의 지역 저장소를 연결한 후 동기화하면 지역 저장소를 인터넷에서 백업할 수 있다.

3. **온라인 개발 툴을 사용할 수 있다.**

    깃허브의 코드스페이스 기능을 사용하여 클라우드에서 소스를 작성하고 편집할 수 있다. 나만의 개발 환경을 설정할 수 있고 깃허브로 push하는 과정을 생략할 수 있다.

4. **협업 프로젝트에 사용할 수 있다.**

    원격 저장소이므로 인터넷만 가능하면 누구나 접근할 수 있고, 깃과 깃허브에서 여러 가지 협업 도구를 제공하므로 깃허브를 사용하면 팀원 여러 명이 하나의 프로젝트를 진행하기 쉽다.

5. **자신의 개발 이력을 남길 수 있다.**

    깃허브에서 소스를 수정하고 오픈 소스에 참여해서 하는 일은 사용자 초기 화면에 날짜별로 모두 기록이 남는다.

6. **다른 사람의 소스를 살펴볼 수 있고, 오픈 소스에 참여할 수도 있다.**

    깃허브에는 전세계 개발자들이 공개해 놓은 소스들이 많다. 이 소스를 얼마든지 내 저장소로 가져와서 분석해 볼 수 있죠. 또한 깃허브에는 깃을 비롯해 웹 개발이나 인공지능, 데이터 과학 등 전 개발 분야에 걸쳐 다양한 오픈 소스가 등록되어 있습니다.

### 2. 지역 저장소를 원격 저장소에 연결하기

    1. 원격 저장소의 HTTPS 주소를 복사한다.
    2. git remote add origin [복사한 주소] //해당 명령으로 원격 저장소에 origin을 추가하겠다고 깃에게 알리는 명령
    3. git remote -v //제대로 연결되어 있는지 확인한다.

### 3. 지역 저장소와 원격 저장소 동기화 하기
**원격 저장소에 파일 올리기 - git push** | ***$ git push*** 명령을 사용하여 스테이징 상태에 있는 파일들을 원격 저장소로 올릴 수 있다.

**원격 저장소에서 직접 커밋하기** | 원격 저장소에서 [Add file] - [Create new file]을 선택하여 원격 저장소에서 직접 커밋할 수 있다.

**원격 저장소에서 커밋 내려 받기 - git pull** | 원격 저장소에 있는 소스 파일을 다른 사용자가 수정하거나 깃허브 사이트에서 직접 커밋하면 지역 저장소와 버전 차이가 생깁니다. 이럴 떄는 원격 저장소와 지역 저장소의 상태를 같게 만들기 위해 원격 저장소의 커밋을 지역 저장소로 가져온다. 이것을 pull한다고 한다.

    $ git pull origin main //해당 명령을 사용하여 origin(원격 저장소)을 지역 저장소의 main 브랜치로 가져올 수 있다.

### 4. 깃허브에 SSH 원격 접속하기

**SSH 원격 접속이란?** | SSH는 secure shell의 줄임말로 보안이 강화된 안전한 방법으로 정보를 교환하는 방식입니다. SSH에서는 기본적으로 프라이빗 키와 퍼블릭 키를 한 쌍으로 묶어서 컴퓨터를 인증한다. 일반적으로는 아이디와 비밀번호를 입력하여 접속을 하는데 SSH 접속 방식을 사용하면 자동 로그인 기능을 통해 번거러움을 줄일 수 있다.

**SSH 키 생성하기** | 홈 디렉터리에서 ssh-keygun 명령을 사용하여 SSH키를 만들 수 있다. 이 때, id_rsa 파일이 프라이빗 키이고, id_rsa.pub 파일이 퍼블릭 키이다.

**깃허브에 퍼블릭 키 전송하기** | SSH 방식으로 접근하려면 먼저 사용자 컴퓨터에 만들어져 있는 퍼블릭 키를 깃허브 서버로 전송한 다음 저장한다.
    
    1. clip 명령을 사용해서 id_rsa.pub 파일의 내용을 복사한다.
    2. 깃허브 웹사이트의 [Settings]를 누르고 Access 카테고리에서 [SSH and GPG keys]를 선택한다.
    3. 새로운 퍼블릭 키를 추가하기 위해 [New SSH key]에서 [Add SSH key]를 클릭해서 SSH 키를 추가한다.

**SSH 주소로 원격 저장소 연결하기**
1. 깃허브에서 저장소를 새로 만들 때 기존의 HTTPS 주소가 아닌 SSH 주소를 복사한다.
2. 홈 디렉터리에 'connect-ssh' 디렉터리를 만든다.
3. git remote add origin [복사한 주소] //기존의 방법과 같다.
